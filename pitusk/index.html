Apa itu PiTUSK

PiTUSK adalah layanan upload resumable berbasis Go untuk perangkat edge dan Raspberry Pi. Tujuan utamanya adalah menjaga upload tetap aman dilanjutkan ketika koneksi putus, proses restart, atau sinkronisasi ke tujuan akhir belum selesai.

  • Create session untuk membuka kontrak upload.
  • Patch chunk untuk menulis data bertahap dengan offset yang tervalidasi.
  • Resume untuk melanjutkan dari posisi terakhir tanpa mengulang dari awal.
  • Finalize + sync untuk memindahkan hasil upload dari staging ke final storage.

Executive Summary

PiTUSK menjaga progres upload tetap aman di lingkungan edge yang tidak stabil, sekaligus menyiapkan jalur adopsi sebagai service maupun framework.

Arsitektur Dasar

API Layer

Mengelola endpoint HTTP, auth token, validasi header, dan routing request ke upload engine.

Upload Engine

Menangani lifecycle sesi: create, write chunk, update offset, finalize, delete, dan validasi status.

Storage Layer

Memisahkan staging file, final file, dan metadata sesi berbasis SQLite agar recovery tetap sederhana.

Sync Engine

Worker background untuk menyalurkan file final ke target lain, dengan fondasi untuk adapter lanjutan.

Mulai Cepat

cd /Users/davidrinaldi/Documents/PROJECT-DAVID/PiTUSK
go run ./cmd/piresum

Alur singkat yang paling sering dipakai:

  1. POST /v1/uploads untuk membuat sesi.
  2. PATCH /v1/uploads/{id} untuk mengirim chunk.
  3. HEAD /v1/uploads/{id} saat perlu resume setelah putus.
  4. POST /v1/uploads/{id}/finalize untuk menyelesaikan sesi.
