PiTUSK Executive Summary (1 Menit)

Ringkasan strategis untuk leadership, product owner, dan sponsor infrastruktur
ExecutiveEdge UploadReliabilityResumable

Inti Narasi

PiTUSK adalah layanan upload resumable untuk lingkungan edge yang menjaga transfer file tetap bisa dilanjutkan walau koneksi putus, proses restart, atau sinkronisasi tertunda.

Nilai utamanya bukan sekadar menerima file, tetapi mengurangi kehilangan progress upload, memudahkan recovery, dan menstandarkan kontrak upload untuk aplikasi lain.

Model operasionalnya: create session → upload chunk → resume via offset → finalize → sinkronisasi lanjutan.

Value Proposition

  • Reliabilitas lebih tinggi: upload tidak perlu mengulang dari awal saat jaringan edge tidak stabil.
  • Recovery terukur: offset dan status sesi disimpan persisten di SQLite.
  • Jejak adopsi jelas: dapat dipakai sebagai service HTTP maupun framework Go publik.
  • Roadmap ekspansif: siap berkembang ke adapter S3/MinIO, retry policy, DLQ, dan observability.

Indikator Kesehatan Strategis

  • /healthz untuk memastikan service hidup.
  • /readyz untuk memastikan service siap menerima trafik.
  • Rasio finalize berhasil vs sesi gagal.
  • Durasi antrean sinkronisasi dan jumlah file tertunda.
  • Lonjakan error offset mismatch atau auth failure.
