Overview

DEMIT (Digital Monitoring Super App) adalah kerangka runtime yang mengelola beberapa application module dalam satu proses. Saat ini fokusnya adalah integrasi aplikasi keamanan IoT PALASIK, dengan arah pengembangan yang bisa ditambah aplikasi berikutnya tanpa merombak core runtime.

Edge Zero Trust Modular Python 3.10+ Research Ready
Status saat ini: DEMIT menampung minimal 1 aplikasi aktif (PALASIK), dan sudah disiapkan untuk ekspansi multi-app.

Aplikasi di dalam DEMIT

1) PALASIK

Decision engine berbasis trust + policy. Bertugas memeriksa event dan menentukan apakah event diproses, dibatasi, atau diblok.

  • Trust Engine (scoring dinamis)
  • Policy Engine (ALLOW / RESTRICT / DENY / QUARANTINE)
  • Plugin registry untuk action extensible
  • Adapter untuk MQTT/HTTP/REST (bergantung konfigurasi)

2) Aplikasi masa depan

DEMIT menyediakan struktur yang sama untuk menambahkan aplikasi lain. Contoh: observability, gateway translasi protocol, dan integrasi enterprise.

  • Routing event berbasis route / app
  • Lifecycle tunggal: start, handle_event, stop
  • Konfigurasi independen per aplikasi

Fitur Core DEMIT

FiturPenjelasan
App runtime Menjalankan beberapa aplikasi dari satu entrypoint CLI.
Routing event Event diarahkan berdasarkan route agar tepat ke aplikasi sasaran.
Isolasi domain aplikasi Konfigurasi dan lifecycle tiap app tidak saling menimpa.
Konfigurasi deklaratif Semua aplikasi didefinisikan dari demit.yaml.
Ekstensibilitas App baru dapat ditambahkan tanpa memodifikasi PALASIK inti.

Fitur PALASIK (Aplikasi Default)

Keamanan

  • Trust score dinamis pada setiap event/device
  • Decision engine untuk enforcement secara real-time
  • Logging keputusan + challenge handler

Operasional

  • Command run dari CLI
  • Support adapter dan plugin action
  • Struktur untuk observability dasar

Riset & Akademik

  • Output keputusan yang mudah ditelusuri
  • Parameter kebijakan bisa didokumentasikan per eksperimen
  • Ringan untuk prototyping dan skripsi/jurnal

Arsitektur Runtime

Konsep alur sederhana: Event → DEMIT Router → Aplikasi target → Enforcement/Output. 

+----------------------+      +----------------------+
|  External Event      | ---> | DEMIT Runtime        |
| (MQTT / HTTP / API)  |      | Router + Dispatcher  |
+----------------------+      +----------+-----------+
                                        |
                                        v
                       +----------------+-----------+
                       | PALASIK App            |
                       | - Trust Engine         |
                       | - Policy Engine        |
                       | - Plugin/Adapter layer  |
                       +------------------------+

Runtime mengelola lifecycle aplikasi dan meminimalkan coupling antar modul.

Quick Start

Contoh minimal agar DEMIT bisa dijalankan lokal:

pip install palasik

demit --config demit.yaml

Contoh demit.yaml:

demit:
  apps:
    - palasik

apps:
  palasik:
    type: palasik
    config_file: "config.yaml"
    plugins_path: "plugins"
    routes:
      - palasik

FAQ Singkat

Apa perbedaan DEMIT dengan PALASIK?

DEMIT adalah super-app runtime/host, sedangkan PALASIK adalah salah satu aplikasi di dalamnya yang menjalankan keputusan keamanan.

Apakah event bisa masuk ke lebih dari satu app?

Ya, jika routing diatur. Jika hanya satu app aktif dan route tidak ditentukan, event bisa diarahkan ke app default.

Apakah dokumentasi ini akan terus dikembangkan?

Ya. Tahap berikutnya adalah menambah halaman setup detail untuk adapter MQTT/HTTP, plugin lifecycle, dan contoh deployment Raspberry Pi.
