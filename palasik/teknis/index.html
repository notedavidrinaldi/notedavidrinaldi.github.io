Arsitektur Ringkas

Flow inti: Adapter/Event Source → DEMIT Router → PALASIK app → Trust Engine & Policy Engine → Enforcement/Output.
KomponenPeranCatatan
DEMIT RuntimeHost dan routing multi-appKonfigurasi dari demit.yaml.
Trust EngineSkoring kepercayaan event/deviceSkor mempengaruhi keputusan kebijakan.
Policy EngineMenilai rule dan menghasilkan keputusanDecision actions: ALLOW/RESTRICT/QUARANTINE/DENY/CHALLENGE.
PluginsEkstensi aksi dan integrasi observabilityLogger dan audit plugin siap dipasang.
Runtime ActionOutput keputusanRoute, blok, alert, dan logging.

Cara Pakai Nyata (3 Menit)

  1. Bikin setup
    palasik init

    Hasil: config.yaml + policy baseline.

  2. Cek sehat
    palasik check --config config.yaml

    Jika PASS: trust + policy siap dipakai.

  3. Simulasi event
    palasik simulate sample-event.json --config config.yaml
  4. Jalankan gateway
    palasik run --config config.yaml
  5. Monitoring
    palasik status --config config.yaml

    Pantau events_allowed, events_denied, alerts.

Deployment Aman

python3 -m palasik.cli.main policy-snapshot --config config.yaml
python3 -m palasik.cli.main policy-deploy-check --config config.yaml --require-allow

Jika status check PASS, lanjut deploy. Jika tidak PASS, stop dan investigasi perubahan policy.

Rollback Cepat

python3 -m palasik.cli.main policy-rollback --config config.yaml --snapshot runs/policy_snapshots/<snapshot>

Langkah pemulihan standar saat KPI anomali: rollback, check, status, lalu evaluasi.

SOP Harian + Siap Deploy

Shift dimulai

  • 1. palasik check
  • 2. palasik status
  • 3. Verifikasi metrik deny, alert, latency

Sebelum deploy

  • 1. policy-snapshot
  • 2. policy-deploy-check --require-allow
  • 3. Simpan hasil untuk audit

Waktu insiden

  • 1. Cari deny spike dan alert berulang
  • 2. Rollback ke snapshot aman
  • 3. Lanjutkan pengecekan rutin

Indikator Kesehatan

  • events_denied_per_min mendadak naik tajam
  • events_allowed_per_min turun drastis tanpa kenaikan volume masuk
  • latency p95 naik di jam normal
  • alert trust mismatch berulang

Jika >=2 indikator menetap >3 interval, jalankan rollback evaluasi.
