Arsitektur

Palasik bekerja di edge gateway: adapter/event source → Agent → Trust Engine → Policy Engine → Plugin System → backend/service.
KomponenPeranCatatan
AdapterTerima eventMQTT/HTTP/Webhook (berbasis plugin adapter).
AgentLifecycle runtime PALASIKMenjalankan trust, policy, plugins, metrik.
Trust EngineSkor 0.0–1.0Hanya menilai, tidak mengambil keputusan akhir.
Policy EngineKeputusanAllow/deny/restrict/challenge/quarantine/monitor.
PluginsEksekusi tindakanLogger, audit, forwarding, alert.

Alur Event

Event masuk → TrustEngine.evaluate(event, context)
             → PolicyEngine.decide()
             → Plugin.on_event()
             → Emit decision + reason_code

Pipeline single direction menjaga jejak observability dan memudahkan audit.

Mulai Cepat (5 menit)

  1. Install
    pip install palasik
  2. Init proyek
    palasik init

    membuat config.yaml baseline.

  3. Cek health
    palasik check --config config.yaml

    Jika PASS berarti pipeline trust + policy valid.

  4. Simulasi event
    cat > sample-event.json <<'EOF_JSON'
{ "version": "1", "event_id": "evt-startup-01", "type": "sensor.sample", "source": { "device_id": "edge-sensor-01", "ip": "192.168.1.10" }, "value": 42 }
EOF_JSON

palasik simulate sample-event.json --config config.yaml
  5. Jalankan agent
    palasik run --config config.yaml
  6. Monitoring status
    palasik status --config config.yaml

    Output berisi events_allowed, events_denied, avg_latency_ms, alerts.

Instalasi & Menjalankan dengan Capture

Bagian ini mengikuti panduan terbaru di repo PALASIK agar orang yang baru pertama kali mencoba bisa mengikuti alur tanpa menebak-nebak.

1. Siapkan environment

python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install palasik

Jika mengembangkan langsung dari source repo, gunakan pip install -e '.[dev]' agar command, test, dan docs update ikut tersedia.

2. Init proyek dan cek health

mkdir palasik-demo
cd palasik-demo
palasik init
palasik check --config config.yaml

3. Simulasikan satu event

cat > sample-event.json <<'EOF_JSON'
{
  "version": "1",
  "event_id": "evt-demo-01",
  "timestamp": "2026-07-03T10:00:00Z",
  "type": "sensor.temperature",
  "source": {
    "device_id": "edge-sensor-01",
    "ip": "192.168.1.10"
  },
  "value": 42
}
EOF_JSON

palasik simulate sample-event.json --config config.yaml

4. Cek status runtime

palasik status --config config.yaml

5. Jalankan agent atau API

palasik run --config config.yaml
palasik serve-api --config config.yaml

Gunakan run saat PALASIK menerima event dari adapter aktif, dan serve-api saat ingin mengekspos PALASIK sebagai service HTTP ringan untuk integrasi lokal.

Konfigurasi dasar

palasik:
  broker:
    host: localhost
    port: 1883
    topic: palasik/sensor/#

  policy:
    version: "1"
    default_deny: true
    default_action: DENY
    policy_id: palasik-baseline
    type: rule
    rules:
      - id: deny_unknown_device
        action: DENY
        reason_code: UNKNOWN_DEVICE
        condition:
          op: eq
          key: source.device_id
          value: unknown
      - id: allow_trusted_device
        action: ALLOW
        reason_code: TRUSTED_DEVICE
        condition:
          op: gte
          key: trust_score
          value: 0.75

Konfigurasi penuh ada di docs/CONFIG.md.

Policy dan Trust

Decision actions yang didukung

  • ALLOW / DENY / MONITOR / RESTRICT / CHALLENGE / QUARANTINE

Contoh rule penting untuk migration path saat ini: gunakan condition (bukan when legacy) dan untuk setiap DENY wajib punya reason_code.

DEMIT integration

demit:
  apps:
    - palasik

apps:
  palasik:
    type: palasik
    config_file: "config.yaml"
    plugins_path: "plugins"
    routes:
      - palasik

Kontrak Health

  • UP → HTTP 200
  • DEGRADED → HTTP 200 atau 503 sesuai mode degraded_http_mode
  • DOWN → HTTP 503

Pemisahan failure yang dipakai sekarang:

  • check-startup dipakai untuk fail-fast sebelum service hidup
  • /health dipakai untuk degradasi runtime setelah service hidup

Shortcut Operasional

make edge-health
make edge-health-wait
make edge-health-strict
make edge-post-restart-check
make edge-post-restart-check-strict

Gunakan helper ini untuk cek health cepat, mode wait saat service baru start, atau gate ketat pasca-restart.

Deployment aman (gate)

palasik policy-snapshot --config config.yaml
palasik policy-deploy-check --config config.yaml --require-allow --max-deny-ratio 0.95 --smoke-events docs/samples/policy-smoke-events.json

Jika gagal (PASS tidak tercapai), hentikan rollout dan evaluasi ulang rule.

Catatan penting: jika Anda menjalankan command dari folder kerja baru, jangan mengandalkan path default smoke events. Berikan --smoke-events secara eksplisit atau salin file sample ke workspace Anda sendiri.

Untuk verifikasi pasca-restart secara praktis, gunakan make edge-post-restart-check atau versi strict-nya.

Rollback

palasik policy-rollback --config config.yaml --snapshot runs/policy_snapshots/<snapshot_file>.snapshot.yaml

Rollback menulis backup ke runs/policy_backups/ lalu mengganti palasik.policy aktif.

Runbook operasional ringkas

Start Shift

  1. palasik check --config config.yaml
  2. palasik status --config config.yaml
  3. Validasi events_denied, alerts, dan status health.

Deploy Policy

  1. snapshot
  2. policy-deploy-check
  3. deploy
  4. 2 menit pertama: check + status

Incident

  1. backup status saat ini
  2. rollback cepat
  3. check ulang dan catat evidence
