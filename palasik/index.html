PALASIK bukan mikrokontroler

PALASIK adalah framework software untuk keamanan IoT berbasis Zero Trust, bukan perangkat keras mikrokontroler.

  • PALASIK: software untuk evaluasi trust, policy, dan enforcement event IoT.
  • Mikrokontroler: hardware seperti ESP32, STM32, atau Arduino.
  • Raspberry Pi: single-board computer yang dapat menjadi host untuk menjalankan PALASIK.
Ringkasnya: PALASIK dapat diintegrasikan dengan perangkat berbasis mikrokontroler, tetapi PALASIK sendiri bukan jenis mikrokontroler.

Executive Summary

PALASIK menilai event IoT berdasarkan trust score dan policy, lalu memutuskan: proses, monitor, atau tolak.

Alur implementasi

  1. palasik init
  2. palasik check --config config.yaml
  3. palasik status --config config.yaml
  4. policy-snapshot + policy-deploy-check sebelum perubahan kebijakan
  5. policy-rollback jika terjadi degradasi

Kontrak Health

Kontrak runtime terbaru yang dipakai PALASIK untuk health endpoint dan helper operasional:

  • UP → HTTP 200
  • DEGRADED → HTTP 200 atau 503 sesuai mode degraded_http_mode
  • DOWN → HTTP 503
  • UNREACHABLE → helper script exit code 4 saat endpoint sama sekali tidak bisa dihubungi

Shortcut yang paling sering dipakai operator:

make edge-health
make edge-health-wait
make edge-health-strict
make edge-post-restart-check
make edge-post-restart-check-strict
