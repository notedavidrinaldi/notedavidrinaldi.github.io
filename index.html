Apa itu PALASIK

PALASIK adalah framework keputusan keamanan untuk node IoT yang dijalankan di gateway (umumnya Raspberry Pi). Framework ini memeriksa perilaku device secara real-time lalu menentukan tindakan pengamanan secara adaptif.

Poin inti: tidak semua data langsung dipercaya. Setiap device dinilai dulu lewat mekanisme trust score, lalu diputuskan aksinya.

Apa itu PiTUSK

PiTUSK adalah framework dan service resumable upload untuk lingkungan edge. Fokusnya adalah memastikan upload file tetap bisa dilanjutkan dengan aman setelah koneksi putus, proses mati, atau perangkat restart.

Poin inti: data tidak dikirim sebagai satu tembakan besar. PiTUSK membaginya menjadi sesi, chunk, offset, finalize, lalu sinkronisasi lanjutan bila dibutuhkan.

Arsitektur Dasar

Edge Gateway Layer

Layer pertama di sisi perangkat fisik:

  • MQTT / HTTP listener
  • Pengumpulan event + metadata perangkat
  • Routing ke aplikasi sesuai route

Trust Engine

Mesin penilai trust:

  • Reputasi device
  • Polanya perilaku traffic
  • Skor kepercayaan dinamis

Policy Engine

Lapisan keputusan keamanan:

  • Allow / restrict / isolate / deny
  • Policy by device, route, dan kondisi sistem
  • Integrasi log dan alert

Alur Runtime

LangkahFungsiOutput
1Event masuk dari transport layerDipetakan ke aplikasi target oleh DEMIT
2Trust Engine mengevaluasi perilakuNilai trust 0.0 - 1.0
3Policy Engine mengeksekusi keputusanAllow / Restrict / Quarantine / Alert
4Eksekusi aksi dipublikasikanLog, webhook, forwarding, atau isolasi

Mulai Cepat

Konfigurasi minimal untuk mencoba PALASIK secara lokal:

pip install palasik
python -m palasik --config ./demit.yaml

Contoh demit.yaml:

demit:
  apps:
    - palasik

apps:
  palasik:
    type: palasik
    config_file: "config.yaml"
    plugins_path: "plugins"
    routes:
      - name: palasik
        path: "palasik.events"

Operasional Cepat

Shortcut Health

make edge-health
make edge-health-wait
make edge-health-strict

Gunakan untuk cek health cepat, mode wait pasca-startup, atau gate ketat yang hanya menerima status UP.

Post-Restart Check

make edge-post-restart-check
make edge-post-restart-check-strict

Menjalankan check-startup lalu verifikasi health berurutan setelah restart atau deploy ringan.

Kontrak Health

  • UP → HTTP 200
  • DEGRADED → HTTP 200 atau 503 tergantung mode
  • DOWN → HTTP 503
  • UNREACHABLE → helper script exit code 4

Deploy Edge

Tahap 1

Siapkan Raspberry Pi + Node-RED (opsional) + MQTT broker internal.

Tahap 2

Aktifkan event source dan route ke app PALASIK sesuai kebutuhan pilot.

Tahap 3

Atur policy, aktifkan log, dan verifikasi tindakan serta kontrak health pada skenario abnormal.

FAQ

Perlu DEMIT agar PALASIK berjalan?

Ya. DEMIT bertugas sebagai runtime super-app yang memproses route event ke PALASIK.

Bisa pakai tanpa plugin tambahan?

Bisa. PALASIK bisa berjalan dengan logger dasar, lalu plugin opsional ditambahkan sesuai kebutuhan observability.

Apakah project ini production-ready?

Fokus saat ini pada dokumentasi riset-prototype yang stabil. Tahapan hardening production sudah dimasukkan ke roadmap berikutnya.
