Apa itu PALASIK

PALASIK adalah framework keputusan keamanan untuk node IoT yang dijalankan di gateway (umumnya Raspberry Pi). Framework ini memeriksa perilaku device secara real-time lalu menentukan tindakan pengamanan secara adaptif.

Poin inti: tidak semua data langsung dipercaya. Setiap device dinilai dulu lewat mekanisme trust score, lalu diputuskan aksinya.

Arsitektur Dasar

Edge Gateway Layer

Layer pertama di sisi perangkat fisik:

  • MQTT / HTTP listener
  • Pengumpulan event + metadata perangkat
  • Routing ke aplikasi sesuai route

Trust Engine

Mesin penilai trust:

  • Reputasi device
  • Polanya perilaku traffic
  • Skor kepercayaan dinamis

Policy Engine

Lapisan keputusan keamanan:

  • Allow / restrict / isolate / deny
  • Policy by device, route, dan kondisi sistem
  • Integrasi log dan alert

Alur Runtime

Quick Start

Konfigurasi minimal untuk mencoba PALASIK secara lokal:

pip install palasik
python -m palasik --config ./demit.yaml

Contoh demit.yaml:

demit:
  apps:
    - palasik

apps:
  palasik:
    type: palasik
    config_file: "config.yaml"
    plugins_path: "plugins"
    routes:
      - name: palasik
        path: "palasik.events"

Deployment

Tahap 1

Siapkan Raspberry Pi + Node-RED (opsional) + MQTT broker internal.

Tahap 2

Aktifkan event source dan route ke app PALASIK sesuai kebutuhan pilot.

Tahap 3

Atur policy, aktifkan log, dan verifikasi tindakan pada skenario abnormal.

FAQ

Perlu DEMIT agar PALASIK berjalan?

Ya. DEMIT bertugas sebagai runtime super-app yang memproses route event ke PALASIK.

Bisa pakai tanpa plugin tambahan?

Bisa. PALASIK bisa berjalan dengan logger dasar, lalu plugin opsional ditambahkan sesuai kebutuhan observability.

Apakah project ini production-ready?

Fokus saat ini pada dokumentasi riset-prototype yang stabil. Tahapan hardening production sudah dimasukkan ke roadmap berikutnya.
