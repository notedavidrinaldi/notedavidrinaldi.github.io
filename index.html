Overview

DEMIT adalah runtime super-app untuk orkestrasi beberapa aplikasi digital dalam satu host tunggal.

Fokus saat ini: PALASIK sebagai aplikasi pertama untuk decision layer keamanan IoT.

Aplikasi di dalam DEMIT

PALASIK

Security app yang mengevaluasi event melalui Trust & Policy Engine, lalu menentukan aksi allowance/restriksi/isolasi.

  • Event trust decision
  • Plugin action
  • Adapter MQTT/HTTP

Future App Slots

Skema yang sama disiapkan untuk menambah app lainnya tanpa mengubah core runtime.

  • Monitoring app
  • Inventory app
  • Dashboard app

Fitur Core DEMIT

FiturDeskripsi
Multi-app runtimeMemuat beberapa aplikasi sekaligus dengan lifecycle yang konsisten.
Route-driven event dispatchMengarahkan event ke target app tertentu melalui route atau auto-default.
Declarative configSemua apps didefinisikan di demit.yaml.
Separation of concernsCore runtime tetap ringan, logika bisnis berada di app layer.

Fitur Utama PALASIK

  • Trust Engine untuk pengukuran trust dinamis.
  • Policy Engine untuk aksi ALLOW / RESTRICT / QUARANTINE / DENY.
  • Plugin model untuk ekstensi log/alert/forwarding.
  • Integrasi ke adapter event (MQTT/HTTP) dan CLI run pipeline.

Detail teknis lebih dalam: halaman dokumentasi PALASIK.

Quick Start

pip install palasik

demit --config demit.yaml

Contoh demit.yaml:

demit:
  apps:
    - palasik

apps:
  palasik:
    type: palasik
    config_file: "config.yaml"
    plugins_path: "plugins"
    routes:
      - palasik

Roadmap

  • Phase 1: runtime DEMIT + satu app PALASIK stabil.
  • Phase 2: command tooling untuk manajemen lifecycle multi-app.
  • Phase 3: UI monitor super-app dan plugin ecosystem.

FAQ

Apa itu route?

Field routing menentukan app tujuan event agar tidak diproses oleh app yang tidak relevan.

Bisa menambah app lain?

Ya, hanya tambah entry di demit.yaml dan registrasi lifecycle app.

Apakah ini production-ready?

Fokus awal adalah riset/prototype yang stabil; production hardening masih dapat dilanjutkan via roadmap phase berikutnya.
