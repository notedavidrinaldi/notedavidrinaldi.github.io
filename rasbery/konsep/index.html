Konsep Dasar Rasbery

Bagian paling penting sebelum instalasi: memahami peran Raspberry Pi.

Arsitektur & peran

Raspberry Pi dipakai sebagai edge node. Ia tidak selalu menggantikan server pusat, tetapi berfungsi sebagai titik eksekusi dekat perangkat fisik.

  • Input: tombol, sensor, scanner
  • Process: validasi lokal, caching, queue
  • Output: logika akses, relay, notifikasi
  • Output API: kirim status ke backend pusat

Dalam konteks ini, modelnya stateful but simple agar mudah dioperasikan.

Bagan mental yang dipakai

  • Device layer (GPIO/sensor)
  • Edge service layer (Python service)
  • API layer (backend)
  • Ops layer (observability)

Setiap layer dipisahkan supaya perubahan bisnis tidak merusak kontrol fisik.
