Operasional Rasbery

Agar tidak sering gangguan saat jam operasional.

Checklist harian

  • systemctl status service utama
  • pemakaian CPU/RAM
  • free storage & rotasi log
  • ping ke backend/API

Fallback plan

Jika jaringan turun:

  1. tetap log event lokal
  2. beri keputusan default aman (DENY)
  3. aktifkan alert via webhook/Telegram

Troubleshoot cepat

  • GPIO diam: cek pin dan grounding
  • API lambat: cek DNS/network timeout
  • service freeze: set restart policy + watch
