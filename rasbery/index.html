Rasbery Library

Panduan komprehensif Raspberry Pi sebagai basis edge device untuk akses kontrol, pengawasan, dan integrasi IoT.
Raspberry Pi Linux GPIO PocongPass Pi Edge Security

1) Apa itu Rasbery?

Rasbery di situs ini dipakai sebagai istilah mudah untuk Raspberry Pi family. Konsep dasarnya: komputer mini berbasis ARM yang hemat daya, stabil, dan fleksibel untuk dijadikan perangkat edge di jaringan lokal, industrial, atau IoT.

Singkatnya, Raspberry Pi tidak hanya untuk coding-mainan. Saat dipasang dengan arsitektur yang tepat, ia bisa menjadi:

  • Gateway Edge (penghubung sensor ke server)
  • Controller Fisik (penggerak relay, sensor, LED, alarm)
  • Node Monitoring (collect metrics, kirim health + status)
  • Mini Server (Web UI lokal, API wrapper, autentikasi cepat)

2) Rasbery bisa digunakan apa saja?

Gunakan sesuai kelas kebutuhan:

Konsep Dasar

  • Arsitektur edge node
  • Peran device, gateway, dan API
  • Mengerti flow sebelum implementasi

Panduan Instalasi

  • Flash OS + setup awal
  • SSH, firewall, dan hardening dasar
  • Bootstrap service edge

Operasional Harian

  • Checklist daily check
  • Troubleshoot cepat GPIO/network
  • Fallback & safety mode

Integrasi Program

  • PocongPass Pi: akses fisik + GPIO
  • Edge Zero Trust Raspberry: pola keamanan layer jaringan
  • PiTUSK: transfer/upload dan reliabilitas koneksi edge

3) Gimana cara install Rasbery?

Instalasi dibagi jadi 5 tahap supaya tidak skip langkah penting:

  1. Siapkan perangkat: Raspberry Pi (Model 4/5 atau CM kalau industrial), catu daya stabil, SSD/SDCard, koneksi jaringan.
  2. Flash OS: gunakan Raspberry Pi Imager (rekomendasi), pilih Raspberry Pi OS Lite (tanpa desktop) untuk server/edge.
  3. Boot pertama: set hostname, user, SSH, timezone, locale, dan update paket.
  4. Hardening: ganti password default, disable login root, setup firewall ringan, update otomatis.
  5. Deploy program: taruh service di path tetap (mis. /opt/pocongpass-pi), lalu setup systemd.

Checklist instalasi cepat (7 menit)

Jalankan:

sudo apt update && sudo apt upgrade -y
sudo apt install -y python3-pip git ca-certificates ufw
sudo systemctl enable --now ssh
sudo apt install python3-venv -y

Dan tetap gunakan SD card grade A/A+ agar lebih tahan write-intensif.

4) Apa yang perlu diketahui untuk mengoperasikan Rasbery?

Biar orang tidak sering bingung, simpan 5 pilar berikut:

Pilar 1 – Sistem Operasi

  • Versi kernel dan OS harus konsisten antar device
  • Update sistem dipadukan dengan jadwal maintenance
  • Gunakan SSH key, bukan password biasa

Pilar 2 – Jaringan

  • Static IP untuk gateway yang selalu dipakai
  • DNS internal yang stabil dan DNS fallback
  • Batasi port terbuka (firewall + policy)

Pilar 3 – Hardware I/O

  • Kenali pinout GPIO sebelum wiring
  • Pastikan ground & daya sesuai rating relay/sensor
  • Tambah optocoupler jika beban tinggi

Pilar 4 – Operasional

  • Logging terstruktur, jangan hanya print
  • Gunakan queue untuk menahan beban burst
  • Timeout + retry wajib untuk request API

Pilar 5 – Keamanan

  • Token di env/secrets, bukan hardcode
  • TLS untuk endpoint internal
  • Audit event + trace id untuk pelacakan

5) Hubungan ke program kita (Rasbery + aplikasi)

Di ekosistem ini, Rasbery adalah platform eksekusi. Program-program kamu memanfaatkan:

  • PocongPass Pi: kontrol akses (scan NIP/API/relay), cocok untuk edge gate.
  • PiTUSK: layanan upload yang tahan putus untuk transfer file operasional.
  • DEMIT / PALASIK: pola runtime, governance, dan struktur dokumentasi yang disiplin.

Strategi koneksi antar program:

  1. Buat satu format trace_id untuk event tap ke open/closed.
  2. Gunakan JSON schema konsisten agar dashboard & AI report bisa membaca lintas program.
  3. Aktifkan health endpoint untuk observability tiap layanan.
  4. Satukan naming service untuk memudahkan support.

7) FAQ Super Lengkap (biar gak sering tanya lagi)

Q: Raspberry Pi mati sendiri saat running relay?

A: Cek power supply (arus dan voltase), cek beban relay, dan jangan lupa ground return yang benar.

Q: Koneksi ke API sering timeout?

A: Cek DNS, MTU, gateway jaringan, lalu naikkan timeout dan retry dengan backoff.

Q: GPIO tidak merespons?

A: Cek mode pin, permission, konfigurasi library, dan jalur fisik kabel.

Q: SD Card cepat rusak?

A: Matikan logging chatty, pindahkan log ke SSD atau rotasi lebih agresif.
